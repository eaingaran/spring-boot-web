Welcome to Jemos Spring Boot Web sample app

The goal os this project is to provide a simple and easy template for a Spring MVC application using Spring Boot. What are the advantages of using Spring Boot?

  • Zero XML configuration for your Spring MVC App
  • Easy Maven setup in a jar project which runs your Web application
  • Full support for unit and integration testing of your webapp without the need to start and stop containers

For years developers have struggled to get an easy way to setup and test a Spring MVC application. Typically one had to:

  • Setup a quite convoluted pom.xml with all necessary Spring dependencies
  • Setup web.xml to register the Spring Dispatcher Servlet
  • Setup Spring application context files, controller setup files, mappings, annotations, etc
  • Come up with all sorts of boilerplate code to just be able to simulate a web request to a web controller with a proper request and response

Although eventually this would work, it would require a lot of effort, especially on the testing front. I'm sure you've been there, googling ways to test a controller in an integration test with a proper request and response.

Now, imagine that your Spring MVC is contained in a Maven jar project, that your controller is a simple Java class with few annotations and that you can exercise it through a fully flagged servlet container like Tomcat or Jetty without having to start anything before your test or shutdown anything after the test has completed.

This represents the holy-grail of web applications speed of development and testing consistency.

Maven setup - it's that easy

First we define the parent project to be spring-boot-starter-parent. This project defines many required dependencies and boilerplate configuration so that we don't have to do the hard work. 

<groupId>uk.co.jemos.experiments.multimodule</groupId>
  <artifactId>spring-boot-web</artifactId>
  <version>1.0-SNAPSHOT</version>
  <packaging>jar</packaging>
  <name>spring-boot-web</name>

  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>1.2.3.RELEASE</version>
  </parent>

  <url>http://mtedone.github.io/spring-boot-web/</url>

  <properties>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <spring.boot.version>1.2.3.RELEASE</spring.boot.version>
  </properties>

We then define the Spring Boot maven plugin and (in my case), Java 8 compiler:

<plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
      </plugin>
      <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-compiler-plugin</artifactId>
        <version>3.3</version>
        <configuration>
          <source>1.8</source>
          <target>1.8</target>
        </configuration>
      </plugin>

For testing, I took the following approach:

  • I want all unit tests to run by default when executing mvn goals from the command line
  • I want all integration tests to run only when specifying a particular Maven profile. This is useful for Continuous Integration environments (e.g. Jenkins) where one might want to run unit tests at each commit and integration tests maybe nightly or a bit less frequently.

For unit tests to execute I define the Maven surefire plugin:

<plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-surefire-plugin</artifactId>
            <version>2.18.1</version>
            <configuration>
                <skip>false</skip>
            </configuration>
            <executions>
                <execution>
                    <id>surefire-it</id>
                    <phase>test</phase>
                    <goals>
                        <goal>test</goal>
                    </goals>
                    <configuration>
                        <skip>false</skip>
                    </configuration>
                </execution>
            </executions>
        </plugin>

For integration tests I specify a dedicated profiles which excludes the unit tests executing only the integration tests through the Maven failsafe plugin. I have also to exclude the execution of unit tests and I achieve this through the Maven surefire plugin configuration specifically stating that I don't want any unit tests to run while executing integration tests.

<profiles>
    <profile>
      <id>integration-tests</id>
      <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <configuration>
                    <skip>true</skip>
                </configuration>
                <executions>
                    <execution>
                        <id>surefire-it</id>
                        <phase>integration-test</phase>
                        <goals>
                            <goal>test</goal>
                        </goals>
                        <configuration>
                            <skip>true</skip>
                        </configuration>
                    </execution>
                </executions>
            </plugin>
          <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-failsafe-plugin</artifactId>
            <version>2.18.1</version>
            <executions>
              <execution>
                <goals>
                  <goal>integration-test</goal>
                  <goal>verify</goal>
                </goals>
              </execution>
            </executions>
          </plugin>
        </plugins>
      </build>
    </profile>
  </profiles>

To execute unit tests I can simply type: 

mvn clean test

Or 

mvn clean install

To execute integration tests I can simply type: 

mvn clean verify -Pintegration-tests
